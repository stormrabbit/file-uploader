## Context

项目是无鉴权的个人文件管理服务，使用 Prisma + SQLite 存储文件元数据，文件实体存放在 `static/YYYY-MM-DD/` 目录。当前没有批量清空数据的手段，调试或重置环境时需要人工操作数据库和文件系统。

## Goals / Non-Goals

**Goals:**
- 提供单个 HTTP 端点，清空所有 DB 文件记录
- 返回可观测的结果（删除记录数）
- 磁盘文件不受影响，防止误触丢失数据

**Non-Goals:**
- 删除磁盘文件（刻意排除，防止误操作）
- 细粒度删除（按日期、按用户等）—— 现有 `DELETE /files/:id` 已覆盖
- 软删除 / 回收站
- 鉴权保护（与全局策略保持一致，局域网部署）

## Decisions

**1. 路由：`DELETE /files/clear-all` 而非 `DELETE /files`**

`DELETE /files` 语义模糊，容易与"批量删除指定列表"混淆。`clear-all` 明确表达全量清除意图，降低误用风险。

**2. 只清空 DB，不动磁盘**

磁盘文件是不可恢复资产，DB 记录可从文件系统重建。只清 DB 可以在调试场景下重置列表视图，同时保留文件本体，误触不会造成实质损失。

## Risks / Trade-offs

- **DB 与磁盘不一致** → 调用后磁盘有孤立文件但 DB 无记录；这是有意设计，重新上传同名文件会重建记录
- **不可逆（DB 侧）** → Swagger 注解中明确说明，调用方自行负责
